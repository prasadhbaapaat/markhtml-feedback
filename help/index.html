MarkHTML Feedback System

A complete guide to sharing documents, collecting feedback, and managing your site.

← Back to Admin Panel

1. Setting up your Documents

The system uses standard Markdown (.md) files. You configure your documents in includes/config.php.

  • Place your `.md` files in the content/ folder (e.g., content/document1.md).
  • Open includes/config.php and add them to the ['content']['documents'] array.
  • The parser automatically splits your document into pages based on the heading level you specify (default is Level 2: ##).

2. Collecting Feedback

Once a document is live, users can browse through the pages. At the bottom of every page, they will find a feedback form.

Users must log in to leave feedback. They can provide their Name, Email, a Rating (e.g., "Looks good", "Needs changes"), and their detailed comments.

Tip: Users can also reply to each other's comments to start a threaded discussion.

3. System Administration

The Admin Panel (admin.php) is your control center. Only users with is_admin set to true in the database or config can access this.

Clear Cache

Whenever you update the raw `.md` files, click Clear Cache to force the system to rebuild the HTML pages so your users see the latest version.

Sync Comments

This powerful tool takes all the comments from the database and injects them directly into your original `.md` files for offline reading or version control.

Reset Comments

When a review cycle is complete, you can permanently wipe all comments from the database to start fresh.

4. Managing Users

From the Admin Panel, you can add users manually or upload a CSV file.

  • CSV Format: Ensure your CSV has headers exactly like: email,name,password,is_admin
  • If a user forgets their password, they can use the "Forgot Password" link on the login page (if email features are enabled).
